# Release Notes Validation Rules

## Mandatory Sections
- [ ] Release Summary with date
- [ ] Version number (semantic versioning)
- [ ] At least one of: New Features, Bug Fixes, or Enhancements

## Content Rules
1. **Version Format**: Must follow X.Y.Z.W pattern
2. **Date Format**: DD Month YYYY (e.g., 3rd May 2024)
3. **Links**: All documentation links must be valid
4. **Issue References**: Bug fixes must reference issue IDs

## Quality Checks
- [ ] No spelling errors
- [ ] Consistent formatting
- [ ] Clear, concise language
- [ ] Technical accuracy verified

## Breaking Changes
- Must have ⚠️ warning emoji
- Must include migration steps
- Must specify affected versions

## Prohibited Content
- ❌ Marketing language
- ❌ Internal jargon
- ❌ Broken links
- ❌ Placeholder text like "TBD"
